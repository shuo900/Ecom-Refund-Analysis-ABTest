# 🛒 基于 Python+SQL+Tableau 的电商大促营销策略异常诊断与 A/B Test 归因分析

> **技术栈**：SQL (多表关联/指标计算) | Python (NLP/统计检验) | Tableau (Pareto建模/多维可视化)

---

## 1. 项目背景

### 一、 业务现状
在情人节唇釉大促中，业务侧采用了**“第2件半价，第3件低至20元”**的激进阶梯降价策略。然而，结算系统底层逻辑采用“总价打折，均价分摊”模式（如买3件实付240元，系统均摊单件80元），导致结算价与宣导价格（20元）严重不符。

### 二、 核心痛点（前后端体验割裂）
* **前端转化受损**：宣导价格与结算页金额无法对齐，产生“虚假宣传”误导感，推高购物车流失率。
* **后端客服承压**：用户因“价格算不明白”产生大量无效进线。在 **Tableau 诊断看板** 中发现，“价格异常”与“优惠门槛咨询”成为占比最高的类目，严重挤占人工处理高价值客诉的资源。

### 三、 项目目标
通过量化“复杂规则”带来的隐性成本（退款与客诉），推动前端表达优化。从“复杂阶梯价”转向“所见即所得”的直降模式，实现降低运营成本与提升转化率的业务闭环。

---

## 2. 项目行动及结果 (STAR 法则)

### T - 任务 (Task)
基于 7W+ 条多源异构数据（订单、退款、客服会话）进行全链路诊断。识别出 **33.7万** 的潜在风险金额，并剖析为何 **情人节退款率（15.4%）** 远高于 **年货节（8.5%）**。

### A - 行动 (Action)
1. **多维数据关联与清洗 (SQL & Python)**: 
   - 使用 SQL 打通 `sales_detail` 与 `chat_for_sql` 数据源。
   - 建立用户 ID 关联模型，定位“咨询价格”与“实际退款”之间的强因果关系。
2. **可视化诊断 (Tableau)**:
   - **核心 KPI 板**：实时监测退款率、咨询占比及 GMV 剪刀差。
   - **Pareto（帕累托）模型**：实锤 78% 的损失源于“价格规则看不懂”。
   - **退款原因双环图**：外环展现大类原因，内环下钻具体痛点，实现维度闭环。
   - **时效面积图**：追踪“危机爆发与平复”的时间脉络，量化客诉响应压力。
   - **横向柱状图**：深度解析“价格异常驱动的咨询暴涨”，锁定受灾最严重的单品与价格段。
   - **归因气泡图**：NLP 提取“凑单”、“算错”、“文字游戏”等高频槽点。
3. **统计验证**: 利用 **Power Analysis** 确定样本量，通过 **Z-test** 执行显著性检验。

### R - 结果 (Result)
* **精准归因**：数据证实近 **38%** 的异常退款直接源于“规则理解成本过高”，成功排除了产品本身的质量风险。
* **业务落地**：管理层采纳复盘报告中的优化策略。在随后的 3.8 大促中废除复杂叠加玩法，转为“单件直降”模式。
* **核心成效**：
  - **转化提升**：实验组转化率提升 **19.9%** ($p = 0.0033$)，具有统计显著性。
  - **降本显著**：策略调整后，价格类咨询占比预计下降 **80%** 以上，爆品退款率回落至 **8%** 的健康水位。
